# London Council Kanban вЂ” Extraction Design

**Date:** 2026-06-06  
**Status:** Approved  
**Source prototype:** `kanban-v2-2.html` (Open Design project `3bc15b09`)

---

## Overview

Extract the London Council issue tracker kanban board from a self-contained HTML prototype into a proper Next.js full-stack application backed by PostgreSQL.

---

## Stack

| Layer | Choice |
|-------|--------|
| Framework | Next.js (App Router) |
| Database | PostgreSQL in Docker (`localhost:5432`) |
| ORM | Drizzle ORM + drizzle-kit |
| Styling | CSS Modules (faithful port of prototype styles) |
| Drag & Drop | Native HTML5 drag API (matches prototype) |
| Auth | None (skipped for now) |

## Data Flow

- **Server Components** (`page.tsx` files) fetch initial data directly from Postgres via Drizzle вЂ” no client-side fetching on load.
- **Mutations** (create ticket, update status, add comment) use **Server Actions** in `lib/actions.ts`.
- **Drag-and-drop reordering** uses `PATCH /api/tickets` вЂ” a lightweight API route, because the reorder event fires from a client drag handler and needs to batch multiple position updates.
- After any mutation, `router.refresh()` re-runs the Server Component data fetch to keep the UI in sync.

## UI & Interactions

Visual design is a faithful port of the prototype:
- Neutral Modern design system
- Cobalt `#2F6FEB` accent, Inter typography, `#FAFAFA` background
- Hairline borders, clean card surfaces

Interactions:
| Feature | Implementation |
|---------|---------------|
| Drag & drop between columns | Native HTML5 drag API в†’ `PATCH /api/tickets` |
| Ticket detail panel | Slide-in drawer, Server Action for status change + comments |
| Create ticket modal | Form в†’ `createTicket` Server Action |
| Search + filter toolbar | Client-side filter on board data already in memory |

## Local Dev Setup

1. `docker compose up -d` вЂ” starts Postgres on `localhost:5432`
2. `npm run db:migrate` вЂ” runs drizzle-kit migrations
3. `npm run db:seed` вЂ” seeds 16 tickets + team members
4. `npm run dev` вЂ” Next.js dev server on `localhost:3000`
